工作流 REST API

通过 REST 接口立即触发任意工作流，支持 Bearer Token 认证

接口信息

Endpoint: POST /api/workflows/:workflowId/trigger

Content-Type: application/json

认证方式: Authorization: Bearer <SERVER_API_KEY>

快速开始

curl -X POST http://localhost:8000/api/workflows/weixin-article-workflow/trigger \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer your-api-key" \
  -d '{
    "payload": { "forcePublish": true },
    "trigger": "api"
  }'

请求参数

参数 位置 类型 必填 说明
workflowId Path string 是 工作流标识，如 weixin-article-workflow
payload Body object 否 传入工作流的参数，默认为空对象
trigger Body "cron" | "api" | "manual" 否 触发来源，默认 manual

支持的工作流

  • weixin-article-workflow — 微信文章工作流
  • weixin-aibench-workflow — AI Bench 工作流
  • weixin-hellogithub-workflow — HelloGitHub 工作流

响应示例

成功

{
  "runId": "run-20241108-001"
}

认证失败

{
  "error": {
    "code": -32001,
    "message": "未授权的访问"
  }
}

无效工作流

{
  "error": {
    "code": -32602,
    "message": "无效的工作流"
  }
}

错误代码

错误代码 含义 解决方案
-32001 未授权访问 确认 Authorization 请求头配置正确
-32602 无效的工作流或参数 检查 workflowId 及请求体格式
-32603 服务器内部错误 查看服务端日志获取更多信息
